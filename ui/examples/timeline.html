Badge ✔ Notification Avatar ✔ Chip ✔

Unread entries use a bg-violet-50 background tint and a filled violet dot overlaid on the Avatar. Read entries use muted zinc text with no dot. Zone 2 is omitted. The vertical spine is replaced by a list-item layout — no spine connector between notification entries.

Badge ✔ Dropdown ✔ Chip ✔ List 🧪

Day group headers ("Today", "Yesterday", "March 10") use a horizontal rule to visually separate clusters. Within each group, the vertical spine connects events as normal. The dot color encodes severity: red (Alert), amber (Warning), emerald (Success), zinc (Info). All 3 zones are required.

Zone 1 · Widget Header Badge · progress label

Asio Neon components used

Badge ✔ Stepper 🧪 Status

Three milestone dot states: filled emerald ✓ (Completed), pulsing violet ring (In Progress), empty ring with zinc border (Upcoming). Upcoming text uses muted zinc tones to communicate the unstarted state. Zone 2 is omitted. This variant is not an event log — do not use the Timeline framework if there are no timestamps or the sequence is user-directed steps (use Form › Stepper instead).

Variant Selection Guide

Variant When to use Zone 2 required? Actor attribution?
Activity Feed Ongoing record activity — tickets, comments, status changes No Mandatory on every entry
Audit Trail Compliance log — who changed what and when Yes — actor filter Always visible — never hover-only
Notification Feed Personal unseen events for the current user No Optional (actor = sender)
Grouped Timeline Multi-day event log needing day-level orientation Optional — event-type filter Recommended
Milestone Timeline Curated past + future milestones (not a raw event log) No Not applicable
