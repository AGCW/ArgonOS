Design Decisions & Evidence
Placement
Top-right nav trigger (always visible). Drawer slides in from the right, overlaying but not replacing the main ticket list.
Marvin #118315: "top-right placement aligns with PSA design patterns and preserves the main filter view." · Marvin #118145: "slide-out panel prevents losing sight of main ticket list."
Tile content
Ticket ID · Subject (truncated to 1 line) · Company · Status chip · Time since last accessed.
PRD spec: ID, subject, company, status required. Time supports recency scanning.
Inaccessible
Degraded tile: lock icon, italic placeholder subject, "Restricted" chip, no-pointer cursor. Preserved in list so technician knows the ticket existed.
PRD risk table; Discovery pack design risk: "Inaccessible tickets create confusion despite disabled state — ensure clear visual pattern and aria-labels."
Open behavior
⚠ Unresolved: PRD defers "drawer stays open/closes on click" to UX decision. This concept shows drawer open after navigation for discoverability — requires PM/UX sign-off.
Discovery pack UX Question: "When the drawer is open and the user clicks a recent ticket, should the drawer stay open or close?"
Hover preview
Not shown (MVP out of scope). Tile hover shows layer-01 background only. Capture hover preview as enhancement backlog item.
Marvin #118275: "Jodie misses hover-over feature to view latest update without clicking in." PRD: out of scope for MVP.
Max count
10 tickets maximum per PRD. No pagination — oldest entry drops off when #11 is accessed. List header communicates "Last 10 accessed."
PRD: fixed at 10. Discovery gap: "ideal count (10 vs. configurable)" flagged for post-MVP.
